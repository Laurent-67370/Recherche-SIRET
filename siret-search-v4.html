🏠 Présentation générale

Cet outil interroge l'API SIRENE officielle (recherche-entreprises.api.gouv.fr, INSEE / data.gouv.fr) pour enrichir et auditer la base fournisseurs ABRAPA dans Sage FRP 1000.

🔍 Recherche unitaire

Recherche un fournisseur par son nom dans SIRENE et affiche jusqu'à 8 résultats avec leur SIRET, adresse, forme juridique et statut.

Utilisation
1
Nom du fournisseur — Saisir une partie du nom suffit. Ex : PHARMAT, Boehringer, SUPER U 67
2
Code postal (optionnel) — Affine les résultats pour les enseignes nationales avec plusieurs établissements
3
Entrée ou clic Rechercher — Résultats classés par pertinence SIRENE
4
Bouton Copier sur chaque SIRET pour le coller directement dans Sage
💡 Astuce — Pour les associations (EHPAD, SSIAD…), ajouter le code postal précise l'établissement local plutôt que le siège national.
📋 Traitement par lot

Lit un fichier CSV, détecte les lignes sans SIRET (ou avec SIRET nul/zéro) et interroge SIRENE pour chacune. Les SIRET déjà renseignés sont conservés sans appel API.

Préparer le fichier CSV
  • Exportez la liste tiers depuis Sage FRP 1000 (menu Fichier → Export)
  • Formats acceptés : CSV (séparateur ; ou ,), encodage ISO-8859-1 ou UTF-8
  • Colonnes minimales : Nom fournisseur + optionnel SIRET + optionnel Code postal
Déroulement
1
Import — Glisser-déposer ou cliquer la zone de dépôt
2
Mapping — Associer chaque colonne (détection automatique). Si la colonne CP contient des références Sage (F1000…) : laisser sur (aucune)
3
Recherche — Cliquer Lancer. Pause possible à tout moment, session sauvegardée automatiquement
4
Export — CSV complet ou Export Sage FRP 1000 (colonnes CT_* prêtes à importer)

✅ Vérifier un SIRET

Valide un numéro SIRET existant (14 chiffres) en interrogeant directement SIRENE. Utile avant une saisie manuelle dans Sage pour s'assurer que l'établissement est bien actif.

Informations affichées
  • Raison sociale officielle SIRENE
  • Statut actif / fermé
  • Adresse complète de l'établissement
  • Code NAF et forme juridique
  • Bouton copier pour récupérer le SIRET formaté
📌 Entrez les 14 chiffres sans espaces ou avec espaces, les deux formats sont acceptés.
📊 Audit base fournisseurs

Compare en masse les SIRET déjà enregistrés dans Sage avec la base SIRENE officielle. Détecte les établissements fermés, les raisons sociales changées et les SIRET invalides.

Formats de fichier acceptés
  • Excel (.xlsx, .xls) — Export direct depuis Sage FRP 1000 ou toute extraction Excel
  • CSV / TXT — Séparateur ; ou ,, encodage ISO-8859-1 ou UTF-8
Export recommandé depuis Sage FRP 1000

Menu Fichier → Liste → Tiers, exporter avec au minimum les colonnes : CT_Num, CT_Intitule, CT_Siret. Le code postal (CT_CodePostal) améliore la comparaison.

❓ Questions fréquentes
Pourquoi certains fournisseurs sont introuvables ?

Plusieurs raisons possibles : nom trop abrégé ou trop différent de la raison sociale officielle, entreprise étrangère (hors registre français), micro-entreprise non inscrite au RCS, ou SIRET effectivement radié. Essayez la recherche unitaire avec différentes formulations.

Le nom trouvé est différent du nom dans Sage — est-ce normal ?

Oui. SIRENE utilise la raison sociale officielle (ex : PHARMAT SAS) alors que Sage peut contenir un nom commercial ou abrégé (ex : PHARMAT). L'outil signale les écarts significatifs avec le statut Nom différent en ignorant les formes juridiques (SAS, SARL…).

La recherche est-elle limitée en volume ?

L'API SIRENE est gratuite et sans clé API. Elle applique un rate limit : l'outil attend automatiquement 380ms entre chaque requête (environ 150 fournisseurs / minute). En cas de dépassement, une pause de 2s est insérée automatiquement.

La session est-elle sauvegardée si je ferme l'onglet ?

Oui pour l'onglet Traitement par lot : la progression est sauvegardée dans le sessionStorage du navigateur. En rouvrant la même session (même onglet ou même navigateur), un bandeau de restauration propose de reprendre. En revanche, fermer complètement le navigateur efface la session.

Peut-on utiliser ce fichier HTML hors connexion ?

Non. Les 4 modules interrogent l'API SIRENE en temps réel. Une connexion internet est indispensable. Le fichier HTML lui-même fonctionne sans serveur (ouverture directe dans le navigateur).
