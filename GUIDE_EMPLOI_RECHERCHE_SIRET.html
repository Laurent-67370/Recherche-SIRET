🏠 Présentation générale

Cet outil interroge l'API SIRENE officielle (recherche-entreprises.api.gouv.fr — INSEE / data.gouv.fr) pour enrichir et auditer la base fournisseurs ABRAPA dans Sage FRP 1000. Il fonctionne comme un fichier HTML autonome, sans installation ni serveur.

🔍 Recherche & Vérification

Recherche un fournisseur par son nom dans SIRENE et affiche jusqu'à 8 résultats sous forme de fiches détaillées avec tous les champs Sage FRP 1000.

Déroulement

Saisir le nom du fournisseur — Une partie du nom suffit. Ex : PHARMAT, Boehringer, SUPER U 67
Code postal (optionnel) — Affine les résultats pour les enseignes nationales
Entrée ou clic Rechercher — Résultats classés par pertinence SIRENE
📋 Copier un champ — Bouton 📋 sur chaque ligne CT_* pour coller directement dans Sage
📋 Tout copier — Copie tous les champs CT_* en une seule fois, prêts à coller dans Excel ou Sage
Sélectionner des cartes — Cliquer sur une carte (ou sa case ☑) pour la sélectionner ; un bandeau apparaît en bas
📥 Exporter la sélection — Fichier Excel avec uniquement les résultats cochés. Le bouton Exporter tout en haut exporte tous les résultats.
Nom ✕ (badge cliquable) — Cliquer le badge dans le champ réinitialise la recherche instantanément

📋 Traitement par lot

Lit un fichier CSV ou Excel, détecte les lignes sans SIRET (ou avec SIRET nul/zéro) et interroge SIRENE pour chacune. Les SIRET déjà renseignés sont conservés sans appel API.

Formats acceptés

  • Excel (.xlsx, .xls) — Export direct depuis Sage FRP 1000
  • CSV / TXT — Séparateur ; ou , auto-détecté, encodage ISO-8859-1 ou UTF-8
  • Colonnes minimales : Nom fournisseur + optionnel SIRET + optionnel Code postal

Déroulement

Import — Glisser-déposer ou cliquer la zone de dépôt
Mapping — Associer chaque colonne (détection automatique des colonnes Sage CT_*). Si le champ CP contient des références Sage (F1000…) : laisser sur (aucune)
Lancer — Pause/reprise possible à tout moment. Session sauvegardée automatiquement toutes les 10 lignes
Export — CSV complet ou Export Sage FRP 1000 (colonnes CT_* enrichies)
StatutSignification
✅ TrouvéSIRET localisé via SIRENE — tous les champs CT_* enrichis
🔵 ConservéSIRET déjà présent dans le fichier source — aucune requête API
❌ IntrouvableAucun résultat — vérifier le nom ou saisir manuellement
⚠️ Session — En cas de fermeture accidentelle de l'onglet, un bandeau de restauration apparaît au rechargement. Fermer complètement le navigateur efface la session batch.

❓ Questions fréquentes

Comment exporter uniquement certains résultats ?

Dans l'onglet Recherche unitaire, cliquez sur une ou plusieurs cartes pour les sélectionner (fond orange + case cochée). Un bandeau sticky apparaît en bas. Cliquez 📥 Exporter la sélection pour générer un fichier Excel ne contenant que les lignes choisies. Le bouton Exporter tout en haut exporte toujours l'ensemble.

L'historique des recherches est-il permanent ?

Oui depuis la v5.0. L'historique est stocké dans le localStorage du navigateur : il persiste entre les sessions, même après fermeture complète du navigateur. Les 50 dernières recherches sont conservées avec leur horodatage. Le bouton Effacer les supprime manuellement.

L'export de la modale établissements exporte-t-il tout ?

Non depuis la v5.0. L'export respecte le filtre actif : si vous êtes sur Actifs uniquement, seuls les établissements actifs sont exportés. Le nom du fichier inclut le filtre appliqué (_actifs ou _fermes).

Peut-on renommer le fichier HTML ?

Oui, sans aucune restriction. Le fichier ne contient aucune référence à son propre nom.

Pourquoi certains fournisseurs sont introuvables ?

Plusieurs raisons possibles : nom trop abrégé ou différent de la raison sociale officielle, entreprise étrangère (hors registre français), micro-entreprise non inscrite au RCS, ou SIRET effectivement radié. Essayez différentes formulations du nom.

Le nom trouvé est différent du nom dans Sage — est-ce normal ?

Oui. SIRENE utilise la raison sociale officielle (ex : PHARMAT SAS) alors que Sage peut contenir un nom commercial abrégé (ex : PHARMAT). L'outil signale les écarts significatifs avec le statut Nom différent en ignorant les formes juridiques (SAS, SARL…).

La recherche est-elle limitée en volume ?

L'API SIRENE est gratuite et sans clé API. Elle applique un rate limit : l'outil attend automatiquement 380 ms entre chaque requête (~ 150 fournisseurs/minute). En cas de dépassement (HTTP 429), une pause de 2 s est insérée automatiquement.

À quoi sert le bouton « Tout copier » ?

Il copie tous les champs CT_* du résultat sous forme d'un tableau champ/valeur (séparateur tabulation), collable directement dans Excel ou un email. Les champs vides sont inclus avec une valeur vide.

À quoi sert la recherche par SIREN ?

Le SIREN (9 chiffres) identifie une entreprise, le SIRET (14 chiffres) identifie un établissement précis. En saisissant un SIREN dans l'onglet Vérifier, l'outil liste tous les établissements (siège + annexes), utile pour les fournisseurs ayant plusieurs sites.

La session batch est-elle sauvegardée ?

Oui pour les onglets Traitement par lot et Audit : la progression est sauvegardée dans le sessionStorage. Un bandeau de restauration apparaît au rechargement. Fermer complètement le navigateur efface la session batch (mais pas l'historique des recherches unitaires).

Peut-on utiliser cet outil hors connexion ?

Non. Les 4 modules interrogent l'API SIRENE en temps réel. Une connexion internet est indispensable.
